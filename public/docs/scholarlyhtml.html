1. Introduction

1.1 Purpose

MyPictures is a web application developed by three students from the College of Computer Science, University of Iaşi.
The current revision is 1.0 (first release), dated 20.06.2022.
The scope of this product is to offer a free-ware tool to manage personal photos on social media platforms: Unsplash and Twitter. The present Software Requirements Specification describes the whole product. The purpose of this document is to specify the requirements for the M-PIC (MyPictures) site. It will present the purpose and features of the web application and what the application is able to do. In addition, this document will present the way this application is intended to be used and will explain some concepts.

1.2 Document Conventions

  • This document follows the IEEE Software Requirements Specification documentation template.
  • Italic bold text is used for defining custom notions.

1.3 Intended Audience and Reading Suggestions

The document is intended for professors, fellow students, developers and non-technical users. Technical notions are used, but not to the extent that non-technical users would find it incomprehensible. For the simple users we recommend reading the User Interfaces and System Features part.

1.4 Product Scope

MyPictures site is an aggregator-type web application through which a user is able to manage the photos he has stored on Unsplash or Twitter. The application can take personal images posted on the mentioned platforms together with comments, ratings or other elements of interest about them. The user can also upload from his local storage new photos to the application and post them in a new tweet on Twitter. The application also provides multi-criteria search function. The images can be edited within the application by applying various CSS filters, defining masks or animations.
The user benefits from having all his photos in one place, viewing other public photos from Unsplash, Twitter or M-PIC database, easy editing and posting.
The objective of our product is purely educational and non-commercial: to help us, the students who developed it, to gain knowledge and experience in Web Technologies.
The ultimate goal of this app is to help sharing and spreading the beauty and joy that are contained in our users photographs.

2. Overall Description

2.1 Product Perspective

M-PIC is a self-contained product developed by three students under the supervision of Associate Professor PhD Sabin-Corneliu Buraga and Lecturer PhD Nicolae Cosmin Vârlan.
M-PIC is an open source project released on the 20th of June 2022.
The major components of the overall system, subsystem interconnections, and external interfaces can be observed in the following C4 diagrams (level 1 and 2):

2.2 Product Functions

Some of the major functions are:

  • create an account
  • contact, info and documentation
  • login
  • manage (view, publish, delete) personal photos
  • multi-criteria search in personal photos
  • connect to social media apps
  • modify M-PIC personal user info
  • edit or animate photos (personal or public)
  • upload photos from local storage (with exif data)
  • post tweet with text and image on Twitter
  • multi-criteria exploratory search on public photos from Unsplash, Twitter or M-PIC

2.3 User Classes and Characteristics

  • Authenticated users who want to simplify the management of their social media photos or explore beautiful photos from other users.
  • Supervisor users that connect via a REST API (with special bearer token) to manage M-PIC user and image data.

2.4 Operating Environment

Can be used on any device with a web browser that supports HTML5, CSS and JavaScript. Media queries allow for a comfortable use on any screen resolution (above 400px width).

2.5 Design and Implementation Constraints

In order to fully use the M-PIC features the user should have an account on Unsplash and/or Twitter. It is good to know that Unsplash does not allow for third party apps like M-PIC to post photos on behalf of authenticated users, but only to retrieve public or personal photos. M-PIC interface is developed using only vanilla HTML5, CSS3, JavaScript without any frameworks or plugins.
The project backend is written in Node JS and is based on MVC (more exactly Model-View-Presenter-ViewModel) architecture that organizes the project in 3 main components : Models, Views and Controllers. The MVC structure makes the application modular and easy to maintain and further develop.

2.6 User Documentation

A video tutorial can be accessed here: https://youtu.be/tZ_KoTgDZAA

2.7 Assumptions and Dependencies

Any change of policy on behalf of Unsplash or Twitter regarding their API for third party apps could affect M-PIC features. M-PIC is developed using JavaScript and HTML5, therefore disabling the JavaScript or using a browser that does not support HTML5 could result in users being unable to use the site. Other issues could occur from our hosting solution: heroku.com and cloud solution: cloudinary.com if their servers might temporary fall.

3. External Interface Requirements

3.1 User Interfaces

3.2 Hardware Interfaces

This product is a web application that does not require any hardware interfaces. It doesn't use hardware acceleration and it works in a browser that has to be installed on the user's device (PC or mobile).

3.3 Software Interfaces

M-PIC requires a browser compatible with HTML5 and CSS3 as well as JavaScript being enabled. In order to take advantage of all the features of our app the client should have an account on Unsplash and/or Twitter.

3.3.1 Postgres Database

M-PIC uses a Postgres Database provided by our hosting solution heroku.com - both freeware. The database has two schemas: "images" and "users". In "users" we store: id, first and last name, email, username, password (encrypted with bcrypt) and unsplash token. In "images" we store id, user id, url, creation date and exif data (if applicable).

3.3.2 Exif data

M-PIC extracts and presents most important exif data from the photos uploaded by the user - if present. In order to do that we use "piexifjs" open license and open source library (Copyright (c) 2014, 2015 hMatoba https://github.com/hMatoba). Also we parse the gps coordinates into real area/location names using the public open API overpass-api.de (provided by OpenStreetMap openstreetmap.org).

3.3.3 Unsplash API

M-PIC is a registered app at Official Unsplash API. Using our credentials and the access granted by our users we can retrieve public photos by search string or user id. The communications are secured with authorization headers containing encrypted bearer tokens.

3.3.3 Twitter API

M-PIC is a registered app at Twitter Developer Platform. Using our credentials, OAuth 1.0a and OAuth 2.0 access protocols we can make specific requests on behalf of authenticated users. We can retrieve public tweets by search string or user id as well as our users profile picture and personal tweets and pictures. The communications are secured with authorization headers containing encrypted bearer tokens or oauth signatures.

3.3.4 Public Images API

M-PIC offers to third party apps a public REST API called "Public Images" at host https://m-pic.herokuapp.com/image where a search string can be passed as a querry parameter: "keyword=searchString" and up to 30 public pictures matching the search from M-PIC database will be retrieved in an array of JSON objects containing url and other useful info.

3.4 Communications Interfaces

M-PIC requires an internet connection to communicate with the web servers for retrieving personal or public photos, connecting to social media and so on.
The communication between a user and the app is done according to the standard protocol HTTPS.

4. System Features

4.1 Photo Manager

4.1.1 Description and Priority

The user can manage all the photos from his social media accounts on Unsplash and Twitter in one place. He can also upload local personal photos to M-PIC for editing and further posting on Twitter.
This feature has a high priority, being the main utility of this site.

4.1.2 Stimulus/Response

  • After registering or logging on our app the user lands on the dashboard page where he can establish connections with third party apps and import his photos (3.1 screenshot).
  • If connections are already established, the user has access to all the photos imported from social media. Also the photos previously stored on M-PIC will be available.
  • Along with the photos M-PIC will import comments, ratings, exif data or other elements of interest.
  • Photos can be individually edited, animated, posted again or made publicly available to other users on M-PIC in the Explore page.

4.1.3 Functional Requirements

  • M-PIC must be able to connect via specific APIs to Unsplash and Twitter. In case of failure the user will be notified.
  • M-PIC must be able to store photos edited inside app for later retrieval to the user.
  • M-PIC must be able to retrieve photos with associated comments, number of likes/shares and exif data.

4.2 Multi criteria search

4.2.1 Description and Priority

From my account or from explore page the user can filter the photos displayed using the search bar (3.1 screenshot). The input can be a string for photo name search or a hash tag for tag search. Using the dropdown navbar the user can filter the photos displayed by their original source, date or number of likes. This feature is medium priority.

4.2.2 Stimulus/Response

  • The user enters a string in the search bar and the displayed photos are reloaded based on their name matching the string.
  • The user selects a social media app from the navbar dropdown and M-PIC shows only the photos imported from that app.
  • The user enters a time interval or number of likes interval and M-PIC shows only that photos which matching the search filters.

4.2.3 Functional Requirements

  • M-PIC must always show the search bar while the user is on explore page or my account page.
  • M-PIC must store available tags with imported pictures.
  • M-PIC must allow time and likes filter.
  • The navbar dropdown must have a form to filter the photos displayed by their original social media app.

4.3 Post picture

4.3.1 Description and Priority

M-PIC makes it easier for users to keep their photos together and upload them to Twitter along with a tweet text. Also M-PIC offers the possibility to make personal pictures publicly available on the explore page. This feature has a high priority as it simplifies the uploading of photos to social media.

4.3.2 Stimulus/Response

  • From the my account page the user can hover over any photo and a quick access twitter icon will appear in the upper right corner of the image. Upon clicking the user will be prompted to enter a message and the image will be posted as a new tweet on Twitter.
  • From the explore page the user can select any picture, enter the editing page, save picture, enter description and then it will be available for posting on Twitter.
  • While in my account page, hovering over a M-PIC image the user will see a locker icon on the lower left corner of the image. Using it the user can enable or disable public or private status of the image.

4.3.3 Functional Requirements

  • M-PIC must provide upload feature for user selected photos via Twitter Developer API.
  • M-PIC must provide upload feature for local stored user photos. Uploaded photos can be made public or private, thus being displayed both in explore and my account pages or only in my account page.

4.4 Photo editor

4.4.1 Description and Priority

M-PIC offers the possibility of editing the photos by applying filters (contrast, blur, brightness, opacity, saturation and so on), masks or geometrical transformations (animations). This feature has a medium priority giving the user a chance for some quick editing before uploading photos to social media.

4.4.2 Stimulus/Response

  • From either the explore or my account page the user can click an image and have a detailed view on it. Above the image view there is an access button to the photo editor.
  • In the editing page the user can save changes, download or restore image (undo changes) hitting the appropriate buttons.

4.4.3 Functional Requirements

  • M-PIC must provide a photo editor page.
  • The photo editor must provide basic css filters, geometrical masks, image rotation and basic animations.

4.5 Photo explorer

4.5.1 Description and Priority

The user has two main views in M-PIC: the personal page and the explore page. In the explore page the user can view photos shared by other users of M-PIC or publicly available on Unsplash and Twitter. Any user can make some of his photos public and they will appear on the explore page. This feature has a medium priority and is providing the users with a source of fresh ideas, beautiful pictures and a place for self-promoting.

4.5.2 Stimulus/Response

  • While on the personal dashboard the user can click on the explore navbar icon (3.1 screenshot) and the explore context will be brought up. The default images loaded will be those matching string search "nature". Any keyword or search filters can be applied and the results will be updated.

4.5.3 Functional Requirements

  • M-PIC must hold for every picture stored the personal/public status.
  • M-PIC must have an explore page where public photos can be viewed.
  • M-PIC must give the user the possibility to share photos with other users making them public.

5. Other Nonfunctional Requirements

5.1 Performance Requirements

M-PIC should optimize page animation, cached data, image loading and database queries such that explore and my account pages will load under 2 seconds.

5.2 Safety Requirements

M-PIC is a software product and no possible loss, damage or harm could result from the use of the product. In order to prevent inappropriate content to be uploaded to the explore context future product releases will implement a `report` function.

5.3 Security Requirements

The users accounts are protected by a login system using bcrypt encryption of the passwords and access tokens in order to maintain their privacy and keep their data safe.
Other security measures that are taken: prevention of SQL-injection attacks, blocking of multiple consecutive requests coming from a user in order to prevent application slowdown and blocking the users that have been reported for inappropriate photo uploads.
