1. Introduction

1.1 Purpose

MyPictures is a web application developed by three students from the College of Computer Science, University of Iaşi.
The current revision is 1.0 (first release), dated 20.06.2022.
The scope of this product is to offer a free-ware tool to manage personal photos on social media platforms: Unsplash and Twitter. The present Software Requirements Specification describes the whole product. The purpose of this document is to specify the requirements for the M-PIC (MyPictures) site. It will present the purpose and features of the web application and what the application is able to do. In addition, this document will present the way this application is intended to be used and will explain some concepts.

1.2 Document Conventions

  • This document follows the IEEE Software Requirements Specification documentation template.
  • Italic bold text is used for defining custom notions.

1.3 Intended Audience and Reading Suggestions

The document is intended for professors, fellow students, developers and non-technical users. Technical notions are used, but not to the extent that non-technical users would find it incomprehensible. For the simple users we recommend reading the User Interfaces and System Features part.

1.4 Product Scope

MyPictures site is an aggregator-type web application through which a user is able to manage the photos he has stored on Unsplash or Twitter. The application can take personal images posted on the mentioned platforms together with comments, ratings or other elements of interest about them. The user can also upload from his local storage new photos to the application and post them in a new tweet on Twitter. The application also provides multi-criteria search function. The images can be edited within the application by applying various CSS filters, defining masks or animations.
The user benefits from having all his photos in one place, viewing other public photos from Unsplah, Twitter or M-PIC database, easy editing and posting.
The objective of our product is purely educational and non-commercial: to help us, the students who developed it, to gain knowledge and experience in Web Technologies.
The ultimate goal of this app is to help sharing and spreading the beauty and joy that are contained in our users photographs.

2. Overall Description

2.1 Product Perspective

M-PIC is a self-contained product developed by three students under the supervision of Associate Professor PhD Sabin-Corneliu Buraga and Lecturer PhD Nicolae Cosmin Vârlan.
M-PIC is an open source project released on the 20th of June 2022.

2.2 Product Functions

Some of the major functions are:

  • create an account
  • contact, info and documentation
  • login
  • manage (view, publish, delete) personal photos
  • multi-criteria search in personal photos
  • connect to social media apps
  • modify M-PIC personal user info
  • edit or animate photos (personal or public)
  • upload photos from local storage (with exif data)
  • post tweet with text and image on Twitter
  • multi-criteria exploratory search on public photos from Unsplah, Twitter or M-PIC

2.3 User Classes and Characteristics

  • Authenticated users who want to simplify the management of their social media photos or explore beautiful photos from other users.
  • Supervisor users that connect via a REST API (with special bearer token) to manage M-PIC user and image data.

2.4 Operating Environment

Can be used on any device with a web browser that supports HTML5, CSS and JavaScript. Media queries allow for a comfortable use on any screen resolution (above 400px width).

2.5 Design and Implementation Constraints

In order to fully use the M-PIC features the user should have an account on Unsplash and/or Twitter. It is good to know that Unsplah does not allow for third party apps like M-PIC to post photos on behalf of authenticated users, but only to retrieve public or personal photos. M-PIC interface is developed using only vanilla HTML5, CSS3, JavaScript without any frameworks or plugins.
The project backend is written in Node JS and is based on MVC (more exactly Model-View-Presenter-ViewModel) architecture that organizes the project in 3 main components : Models, Views and Controllers. The MVC structure makes the application modular and easy to mantain and further develop.

2.6 User Documentation

A video tutorial (Youtube link) for users will be added soon.

2.7 Assumptions and Dependencies

Any change of policy on behalf of Unsplah or Twitter regarding their API for third party apps could affect M-PIC features. M-PIC is developed using JavaScript and HTML5, therefore disabling the JavaScript or using a browser that does not support HTML5 could result in users being unable to use the site. Other issues could occur from our hosting solution: heroku.com and cloud solution: cloudinary.com if their servers might temporary fall.

3.2 Hardware Interfaces

This product is a web application that does not require any hardware interfaces. It doesn't use hardware acceleration and it works in a browser that has to be installed on the user's device (PC or mobile).

3.3 Software Interfaces

M-PIC requires a browser compatible with HTML5 and CSS3 as well as JavaScript being enabled.
In order to take advantage of all the features of our app the client should have an account on Unnsplash and/or Twitter. M-PIC will use software interfaces like Twitter API v2 or Instagram Basic Display API to let users import their photos from social media, connect their profiles, and more.
The site can also send e-mails to users for resetting password, confirming email address, or notifying about important events on the site.

4. System Features

4.1 Social Media Photo Manager

4.1.1 Description and Priority

The user can manage all the photos from his social media accounts in one place. For the first release Instagram, Twitter and Reddit will be supported.
This feature has a high priority, being the main utility of this site.

4.1.2 Stimulus/Response

  • After registering or logging on our app the user lands on the dashboard page where he can establish connections with third party apps and import his photos (3.1 screenshot).
  • If connections are already established, the user has access to all the photos imported from social media.
  • Along with the photos M-PIC will import comments, ratings or other elements of interest.
  • Photos can be individually edited, posted again or made publicly available to other users.

4.1.3 Functional Requirements

  • M-PIC must be able to connect via specific APIs to Instagram, Twitter and Reddit. In case of failure the user will be notified and the reason of failure will be provided.
  • M-PIC must be able to retrieve photos with associated comments, number of likes/shares and metadata.

4.2 Multi criteria search

4.2.1 Description and Priority

From my account or from explore page the user can filter the photos displayed using the search bar (3.1 screenshot). The input can be a string for photo name search or a hash tag for tag search. Using the dropdown navbar the user can filter the photos displayed by their original source: Instagram, Twitter or Reddit. This feature is medium priority.

4.2.2 Stimulus/Response

  • The user enters a string in the search bar and the displayed photos are reloaded based on their name matching the string.
  • The user selects a social media app from the navbar dropdown and M-PIC shows only the photos imported from that app.
  • The user enter a string preceded by a hash and M-PIC shows only that photos which are tagged with the given string.

4.2.3 Functional Requirements

  • M-PIC must always show the search bar while the user is on explore page or my account page.

  • M-PIC must store available tags with imported pictures.

  • M-PIC must allow user to add tags to pictures.

  • The navbar dropdown must have shortcuts to filter the photos displayed by their original social media app.

4.3 Photo upload

4.3.1 Description and Priority

M-PIC makes it easier for users to keep their photos together and upload them to preferred apps via quick access links along with a message/description. This feature has a high priority as it simplifies the uploading of photos to multiple social media accounts.

4.3.2 Stimulus/Response

  • From the explore or my account page the user can hover over any photo and a quick access dropdown menu with the three available social media apps will appear in the upper right corner of the image. Upon clicking on one of them the image will be uploaded to the specific social media app.
  • In the explore page the navbar dropdown has a shortcut to the upload feature. The user can select the photos to be uploaded and the desired destination app.

4.3.3 Functional Requirements

  • M-PIC must provide upload feature for user selected photos via third party APIs.
  • M-PIC must provide upload feature for local stored user photos. Uploaded photos can be made public or personal, thus being displayed both in explore and my account pages or only in my account page.

4.4 Photo editor

4.4.1 Description and Priority

M-PIC offers the possibility of editing the photos by applying filters (contrast, blur, brightness, opacity, saturation and so on), masks, tags or geometrical transformations. This feature has a medium priority giving the user a chance for some quick editing before uploading photos to social media.

4.4.2 Stimulus/Response

  • From either the explore or my account page the user has quick access to the photo editor via the dropdown shortcut that appears on photo hovering.
  • In the explore context the user can access the navbar dropdown which has a link to the photo editor.
  • After editing the photo the user can save changes hitting the Finish button.

4.4.3 Functional Requirements

  • M-PIC must provide a photo editor page.
  • The photo editor must provide basic css filters, geometrical masks, tag editing and image rotation.

4.5 Photo explorer

4.5.1 Description and Priority

The user has two main views in M-PIC: the personal page and the explore page. In the explore page the user can view photos shared by the developers and by other users. Any user can make some of his photos public and they will appear on the explore page. This feature has a medium priority and is providing the users with a source of fresh ideas, beautiful pictures and a place for self-promoting.

4.5.2 Stimulus/Response

  • While on the personal dashboard the user can click on the explore navbar icon (3.1 screenshot) and the explore context will be brought up.

4.5.3 Functional Requirements

  • M-PIC must hold for every picture stored the personal/public status.
  • M-PIC must have an explore page where public photos can be viewed.
  • M-PIC must give the user the possibility to share photos with other users making them public.

5. Other Nonfunctional Requirements

5.1 Performance Requirements

M-PIC should optimize page animation, cached data, image loading and database queries such that explore and my account pages will load under 1.5 seconds.

5.2 Safety Requirements

M-PIC is a software product and no possible loss, damage or harm could result from the use of the product. In order to prevent inappropriate content to be uploaded to the explore context future product releases will implement a `report` function.

5.3 Security Requirements

The users accounts will be protected by a login system using an appropriate authentication encryption method in order to maintain their privacy and keep their data safe.
Other security measures that will be taken are : prevention of SQL-injection attacks, blocking of multiple consecutive requests coming from a user in order to prevent application slowdown and blocking the users that have been reported for inappropriate photo uploads.
